아키텍처 개요Cloudera Docs 아키텍처 개요 Cloudera Machine Learning(CML)은 CDP 플랫폼의 데이터 서비스입니다. CDP 플랫폼에 상주하면 CML은 플랫폼에 상주하는 다른 데이터 서비스(예: 데이터 엔지니어링 및 데이터 웨어하우스)를 통합하고 활용하는 데 도움이 됩니다. ProvisioningCML은 CDP 제어 평면을 활용하여 데이터 서비스를 관리하므로 CML 작업 영역을 프로비저닝하고 삭제할 수 있습니다. CDP 제어 평면은 클라우드 관리형 Kubernetes(K8s)와 함께 CPU, 메모리 및 GPU 리소스에 동적으로 액세스하여 인프라를 제공하는 클라우드 네이티브 기능을 활용합니다. CML 아키텍처CML 작업 영역이 프로비저닝되면 엔드투엔드 Machine Learning 워크플로에 Cloudera Machine Learning(CML)을 사용할 수 있습니다. ML 런타임ML 런타임은 데이터 과학 워크로드를 실행하고 기본 클러스터에 대한 액세스를 중개하는 역할을 담당합니다. Kubernetes의 Spark Cloudera Machine Learning(CML)에서는 런타임 추가 기능을 통해 여러 Spark 버전을 사용할 수 있습니다. 데이터 과학자는 모든 워크로드에 대해 구성할 Spark 버전을 선택할 수 있습니다. CML은 런타임 컨테이너를 구성하고 모든 종속성을 마운트합니다.
