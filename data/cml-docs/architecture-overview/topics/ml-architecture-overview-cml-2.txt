CML은 사용자 코드, 세션에 설치된 라이브러리 및 작은 데이터 파일을 포함하는 사용자 프로젝트 파일을 저장하기 위해 NFS 서버, 즉 POSIX 호환 파일 시스템을 사용합니다. AWS의 경우 CML은 프로젝트 파일을 저장하기 위한 작업 공간을 프로비저닝할 때 EFS(탄력적 파일 시스템) 파일 시스템을 생성합니다. Azure의 경우 사용자는 프로젝트 파일을 저장하기 위해 CML 작업 영역을 프로비저닝할 때 Azure NetApp Files에서 생성된 NFS 볼륨 또는 외부 NFS 서버를 제공할 수 있습니다. 이 NFS 서버는 CML 작업공간에 Kubernetes 영구 볼륨(PV)으로 제공됩니다. 영구 볼륨 클레임(PVC)은 네임스페이스별로 설정되며 각 사용자는 자신의 네임스페이스를 갖습니다. 따라서 NFS 서버에 대한 각 사용자의 보기는 PVC에 의해 노출된 보기로 제한됩니다. 상위 주제: 아키텍처 개요
