CDP 제어 평면은 클라우드 관리형 Kubernetes(K8s)와 함께 CPU, 메모리 및 GPU 리소스에 동적으로 액세스하여 인프라를 제공하는 클라우드 네이티브 기능을 활용합니다. 프로비저닝 중에 기계 학습 애플리케이션은 고객의 ID 공급자에 다시 "연결"되는 CDP ID 공급자를 통해 서비스(ML)의 최종 사용자를 인증하도록 구성됩니다. 결과적으로 Machine Learning 프로비저닝된 인스턴스는 원활한 고객 SSO를 허용합니다. CML 작업 영역을 프로비저닝하면 다음과 같은 일이 발생합니다. CDP 제어 플레인은 클라우드 환경에서 다음을 수행합니다. cloudera.site 도메인을 사용하여 TLS 인증서 및 도메인 이름을 요청합니다. SSO 구성을 식별합니다. 환경에 대한 SDX 구성을 식별합니다. CDP 제어 플레인 프로비저닝 관리형 Kubernetes 클러스터 CDP 제어 평면은 CML을 Kubernetes 환경에 설치합니다. 스토리지는 관리형 서비스 공급자를 통해 직접 마운트됩니다. CML은 클라우드 공급자 로드 밸런서와 네트워킹 인프라를 사용하여 리소스를 분할합니다. CML은 또한 클라우드 공급자 인프라를 활용하여 고객이 자동 크기 조정을 지정할 수 있도록 합니다. CML은 DNS와 인증서를 프로비저닝합니다. CML은 지속적으로 고객의 인증서를 갱신합니다. 상위 주제: 아키텍처 개요
