CML 아키텍처Cloudera Docs CML 아키텍처 CML 작업 영역이 프로비저닝되면 엔드투엔드 Machine Learning 워크플로에 Cloudera Machine Learning(CML)을 사용할 수 있습니다. CML은 프레젠테이션 계층, 애플리케이션 계층 및 데이터 계층으로 구성된 3계층 애플리케이션입니다. 웹 계층 CML은 데이터 과학자를 위한 워크로드 및 리소스 관리 작업을 단순화하는 UI를 제공하는 웹 애플리케이션입니다. 이는 사용자에게 분석 파이프라인을 배포 및 확장하고 안전하게 구획화된 환경에서 동료와 협업할 수 있는 편리한 방법을 제공합니다. CML은 HTTPS, Websocket 및 gRPC를 사용하여 통신합니다. 외부 통신은 웹 UI 및 API에 대한 HTTP 및 Websocket으로 제한됩니다. 클러스터 내 서비스 간 통신은 gRPC를 사용하며 TLS(전송 계층 보안)를 사용하여 암호화되고 상호 인증됩니다. 애플리케이션 계층 애플리케이션 계층은 사용자가 실행 중인 실제 워크로드 세트를 사용합니다. 이러한 워크로드는 Kubernetes 네임스페이스에 격리되어 있으며 특별히 표시된 컴퓨팅 노드에서 실행됩니다. Kubernetes/노드 수준 자동 확장은 사용자 요구에 따라 클러스터 크기를 확장/축소하는 데 사용됩니다. 사용자 코드는 S2I(Source-to-Image Pod) 세트를 통해 Docker 이미지에 구워집니다. 여기에는 관리 서버, 빌드용 대기열, Docker용 이미지를 제공하는 레지스트리, git 서버 및 실제로 이미지 구축을 수행합니다. 전통적으로 이러한 이미지는 호스트 시스템의 도커를 사용했지만 CML은 일부 플랫폼의 보안 및 규정 준수를 위해 컨테이너 내 빌드로 전환했습니다. 데이터 계층 CML은 세션, 작업, 모델 및 애플리케이션과 같은 사용자 워크로드의 메타데이터를 유지하기 위해 내부 Postgres 데이터베이스를 사용합니다. 이는 클라우드 제공업체의 블록 스토리지 제품(예: EBS)을 사용하여 포드로 실행되고 영구 볼륨으로 지원됩니다.
