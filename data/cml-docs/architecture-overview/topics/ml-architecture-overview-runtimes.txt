ML RuntimesCloudera Docs ML 런타임 ML 런타임은 데이터 과학 워크로드를 실행하고 기본 클러스터에 대한 액세스를 중개하는 역할을 담당합니다. CML을 사용하면 대화형 세션, 예약된 작업, 배포된 모델이나 애플리케이션을 통해 모든 코드를 실행할 수 있습니다. 데이터 과학자는 대화형 세션을 사용하여 데이터를 탐색하거나 모델을 개발할 수 있습니다. 작업을 생성하고 지정된 시간에 실행되도록 예약하거나 작업을 REST 엔드포인트를 제공하는 모델 또는 비즈니스 사용자를 위한 대화형 데이터 대시보드를 제공하는 애플리케이션으로 생산할 수 있습니다. 이러한 모든 워크로드는 Kubernetes 위에 있는 ML 런타임 컨테이너 내에서 실행됩니다. Cloudera ML 런타임은 특정 사용 사례를 지원하기 위해 특별히 제작되었습니다. 단일 편집기(예: Workbench, Jupyterlab)와 함께 사용할 수 있고 단일 언어 커널(예: Python 3.8 또는 R 4.0)을 제공하며 UNIX 도구 및 유틸리티 또는 언어 라이브러리 및 패키지 세트가 있습니다. ML 런타임은 오픈 소스로 제공되었으며 cloudera/ml-runtimes GitHub 리포지토리에서 사용할 수 있습니다. 런타임 환경을 완전히 이해해야 하거나 처음부터 새 런타임을 빌드하려는 경우 이 리포지토리에서 ML 런타임 컨테이너 이미지를 빌드하는 데 사용된 Dockerfile에 액세스할 수 있습니다. 대부분의 데이터 과학 사용 사례를 포괄하는 기본적으로 지원되는 다양한 런타임이 있지만 사용자 지정 ML 런타임 컨테이너 이미지를 구축하면 특별한 요구 사항을 충족할 수 있습니다. CML은 또한 CPU, GPU 및 메모리에 대한 할당량 관리를 지원하여 사용자가 CML 작업 영역 내에서 액세스할 수 있는 리소스의 양을 제한합니다. ML 런타임 이전에 CML은 레거시 엔진을 통해 유사한 기능을 제공했습니다. 이러한 더 이상 사용되지 않는 컨테이너 이미지는 모놀리식 아키텍처를 따랐으며, 모든 커널(Python, R, Scala), 겉보기에 유용해 보이는 모든 패키지 및 라이브러리가 이미지에 포함되었습니다. 상위 주제: 아키텍처 개요
