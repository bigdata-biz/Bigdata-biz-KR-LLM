ML 작업 영역은 유연하고 확장 가능한 엔진을 통해 Python, R, Scala 및 Spark 워크로드의 완전히 컨테이너화된 실행을 지원합니다. AI 애플리케이션 분석 애플리케이션은 CML 프로젝트 내에서 장기 실행 애플리케이션을 호스팅할 수 있는 장소를 제공합니다. CML은 데이터 과학자가 고급 분석 및 모델을 프로덕션에 수행할 수 있는 장소를 제공하는 반면, 분석 애플리케이션은 CML 프로젝트 내에서 장기 실행 애플리케이션을 호스팅할 수 있는 장소를 제공합니다. 이를 통해 더 많은 사용자 그룹이 CML에서 발견된 통찰력을 얻을 수 있습니다. Flask 및 Streamlit과 같은 다양한 프레임워크를 사용하여 애플리케이션을 구축할 수 있습니다. 시간 초과를 방지하고 ML 런타임을 활용하는 격리된 컴퓨팅 인스턴스 내에서 실행됩니다. 사용자는 웹을 통해 애플리케이션에 액세스할 수 있습니다. 애플리케이션은 대화형 데이터 시각화 호스팅 또는 CML의 배포 모드에 대한 UI 프런트엔드 제공과 같은 다양한 사용 사례에 사용될 수 있습니다. 탐색적 데이터 과학 CML을 사용하면 데이터 실무자가 단일 사용자 인터페이스 내에서 데이터 세트를 검색하고 쿼리하고 쉽게 시각화할 수 있습니다. 모든 데이터 과학 프로젝트의 시작은 필요한 데이터를 찾고 이해하는 것에서 시작됩니다. CML은 탐색적 데이터 분석에 필요한 모든 도구를 단일 UI에 함께 제공하므로 데이터 실무자가 애플리케이션 간에 이동할 필요가 없고 IT가 여러 도구를 지원할 필요가 없습니다. CML은 사용자에게 연결할 수 있는 사용 가능한 데이터 자산 보기, 해당 데이터 소스를 쿼리할 수 있는 SQL 편집기, 데이터를 이해하고 통찰력을 전달할 수 있는 사용하기 쉬운 드래그 앤 드롭 시각화 도구를 제공합니다.
